First things first

Centralized Systems

  • Single Point of Failure
  • Can not work offline
  • Politics

Distributed Systems

  • Every working directory contains the entire history
  • Every checkout is really a full backup of all the data
  • Can do everything offline
  • Sharing with multiple repositories

Git brief history

actually this is very brief ;)

  • Was initially designed and developed by Linus Torvalds for Linux kernel development
  • Torvalds wanted a distributed system that he could use like BitKeeper
  • Very strong safeguards against corruption, either accidental or malicious
  • Very high performance

Git Basics

  • Each Git repository contains a .git folder
  • To clone a repo means you clone it :)
  • To commit means to save some changes to your repo
  • To pull means to update your repo with changes from remote one
  • To push means to update a remote repo with your changes

Git Branches

This is what make Git awesome if you ask me

    $ git branch
    * master
    $ git branch test
    $ git checkout test
    $ git branch
      master
    * test

Basic workflow

    $ cd awesome_project
    $ git checkout -b mybranch
    $ gvim hello_world.rb
    $ git status
    $ git add hello_world.rb
    $ git status
    $ git commit -m "added the hello_world class file"
    $ git checkout master
    $ ls
    $ git merge mybranch
    $ ls
